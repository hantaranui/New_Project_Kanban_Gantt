🔔 Notifications & e-mail

Prévenir les utilisateurs concernés et, en option, envoyer un e-mail via un webhook

📧 Recevoir un e-mail à chaque notification (webhook Grist → n8n)

Le widget enregistre une ligne dans la table PM_Notifications pour chaque personne concernée par une tâche (création ou modification). Un webhook Grist posé sur cette table peut transformer chaque notification en e-mail, via n8n (gratuit / auto-hébergeable).

⬇️ Le plus simple : téléchargez un workflow prêt à l'emploi et importez-le dans n8n (Menu → Import from File), selon votre messagerie :
📧 Gmail 📧 Outlook / Microsoft 365 📧 SMTP (Zimbra & autres)

🏢 Pas de n8n ? Si votre organisation est sur Microsoft 365, suivez plutôt le guide Power Automate (pas-à-pas) ↗ — pas d'outil à installer.

Comment utiliser le fichier .json dans n8n :

  1. Importez le fichier : dans n8n, menu ⋮ (en haut à droite) → Import from File → sélectionnez le .json téléchargé. Le Webhook, la normalisation et l'envoi d'e-mail apparaissent, déjà reliés et mappés sur User_Email, Type, Message.
  2. Ouvrez le nœud « Envoyer l'e-mail (SMTP) » : associez vos identifiants SMTP et votre adresse expéditeur.
  3. Activez le workflow, puis copiez l'URL de production du nœud Webhook.
  4. Dans Grist : ouvre le document → menu ⚙️ Paramètres du document → section Webhooks → Ajouter un webhook :
    • Table : PM_Notifications
    • Événement : Add (à chaque nouvelle notification)
    • URL : colle l'URL du webhook n8n
    • Colonnes (optionnel) : User_Email, Message, Type, Task_Id
  5. Teste : crée ou modifie une tâche concernant un utilisateur → une ligne PM_Notifications est créée → Grist appelle n8n → l'e-mail part. ✅

ℹ️ Les webhooks Grist nécessitent un plan/instance qui les autorise (sur Grist auto-hébergé : variable GRIST_ALLOWED_WEBHOOK_DOMAINS). Le widget ne peut pas envoyer d'e-mail directement (bac à sable navigateur) — c'est pour cela qu'on passe par un webhook.
